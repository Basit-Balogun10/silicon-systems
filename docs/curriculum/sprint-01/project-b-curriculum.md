# Project B Curriculum - Traffic Light Controller FSM (Living)

## Goal

Achieve deterministic FSM design reasoning with explicit transition guards, safety invariants, and timing-aware verification.

## Core Path (A to Z)

1. S01 - Baseline Calibration: establish confidence baseline and assumptions.
2. S02 - FSM Foundations: state, guard, transition, and output decode semantics.
3. S03 - Transition Determinism: legality, priority, and deterministic selection.
4. S04 - Timer-Guard Integration: done-signals as transition enablers.
5. S05 - Architecture Decomposition: state diagram and control/data flow interpretation.
6. S06 - Safety Invariant Design: mutual exclusion and legal-state constraints.
7. S07 - Verification Strategy: transition coverage and randomized sensor patterns.
8. S08 - RTL Structure Review: state encoding and guard implementation choices.
9. S09 - Branch A Closure Evidence: timing and synthesis expectations.
10. S10 - Branch B Closure Evidence: physical-flow expectations.
11. S11 - Final Readiness Review: failure-mode audit and completion check.

## Adaptive Branch Rules

### Branch R1 - Guard Logic Remediation

Trigger:

- Ambiguous or conflicting transition reasoning.

Action:

- Build explicit transition truth table and priority order.
- Re-test transition narrative before progression.

### Branch R2 - Safety Invariant Remediation

Trigger:

- Inability to explain or assert mutually exclusive outputs.

Action:

- Add assertion-first exercise for output safety.

### Branch D1 - Deep-Dive Track

Trigger:

- Strong early mastery.

Action:

- Explore state encoding tradeoffs and timing impact.
